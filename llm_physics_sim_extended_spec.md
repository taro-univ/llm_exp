# LLM推論崩壊・非線形力学系モデリング拡張プロジェクト 仕様書

Appleの研究論文の知見と、既存の1次元ポテンシャル井戸シミュレータの構造をベースに、非線形力学系を用いた最先端のLLM推論メカニズムの解明および制御システムを構築するための仕様を定義します。

## 1. 評価環境（パズル）の完全実装
現在の単一パズル環境から、複雑度を厳密に制御可能なテストベッドへとシステムを拡張します。

* **新規環境クラスの追加**: 現在実装されている `TowerOfHanoiEnv` クラスに加え、論文で検証された3つのパズル環境のシミュレータクラスを実装します。
    * **CheckerJumpingEnv**: 赤と青のチェッカーを入れ替える1次元パズル。複雑度はチェッカー数 $N$ に対して二次関数的に増加します。
    * **RiverCrossingEnv**: 安全制約（他のエージェントとクライアントの同席禁止など）を満たしながら川を渡る計画問題。複雑度はペア数 $N$ とボート定員 $k$ で管理します。
    * **BlocksWorldEnv**: ブロックのスタックを目標の配置に並べ替える問題。ブロック数 $N$ で複雑度を管理します。
* **main関数の改修**: ユーザーが実行時に「パズルの種類」「問題の複雑度 ($N$)」「検証するAIモデル」を選択できるように、main関数および実行時引数を拡張します。

### 各パズルの詳細仕様
#### ① チェッカージャンプ (Checker Jumping)
赤と青のチェッカーを入れ替える1次元の制約充足パズルです。
* **初期状態と目標**: 左に $N$ 個の赤（'R'）、中央に1つの空きマス（'_'）、右に $N$ 個の青（'B'）が並んだ状態（長さ $2N+1$）からスタートし、赤と青の配置を完全に反転させるのが目標です。
* **ルール**:
    * 隣の空きマスへ前進する「スライド」。
    * 違う色のチェッカーを1つだけ飛び越えて空きマスに着地する「ジャンプ」。
    * チェッカーは後ろに下がることはできない（赤は右へ、青は左へしか進めない）。
* **シミュレータの実装要件**: 盤面の境界チェック、指定されたマスに正しい色のチェッカーがあるか、移動先が空いているかを確認します。さらに、移動距離が1（スライド）か2（ジャンプ）か、正しい方向に進んでいるか、ジャンプの場合は飛び越えるマスに反対色のチェッカーがいるかを多層的に検証する必要があります。

#### ② 川渡り (River Crossing)
$N$ 人のアクターと対応する $N$ 人のエージェント（護衛など）が、全員で左岸から右岸へボートで渡る計画問題です。
* **ルールと制約**:
    * **ボートの定員制約**: ボートの定員は $k$ 人（$N \le 3$ の場合は2人、それ以上の場合は3人）です。
    * **無人ボートの禁止**: ボートは最低1人乗っていないと動かせません。
    * **安全制約（最重要）**: 「自分のエージェントが不在の状態で、他のエージェントと同じ場所にいてはいけない」。これは両岸だけでなく、ボートの上でも適用されます。
* **シミュレータの実装要件**: ボートの位置（左岸・右岸）をターンごとに切り替えながら、各個人の状態を追跡します。移動のたびに定員チェックを行い、移動後には「左岸」「右岸」「ボート上」のすべてで安全制約が破られていないかを厳密に判定してスコア化します。

#### ③ ブロックの世界 (Blocks World)
積まれたブロックを特定の順序に並べ替える、古典的な計画タスクです。
* **初期状態と目標**: 論文の実験では、$N$ 個のブロックがアルファベット順に2つのスタックに分割された状態から始まります。目標状態は、それらを1つのスタックに交互に（インターリーブして）積み直すという複雑な構成になっており、スタックの完全な解体と再構築が必要になります。
* **ルール**:
    * どのスタックでも一番上（トップ）にあるブロックしか動かすことができない。
    * ブロックは、空の場所か、他のブロックの一番上のいずれかにしか置けない。
* **シミュレータの実装要件**: 指定されたスタックのインデックスが範囲内か、移動元のスタックが空でないかをチェックします。最も重要な検証として、「動かそうとしているブロックが本当にそのスタックのトップにあるか」を確認し、問題なければリストの `pop` と `append` で状態を更新して、最終的に目標のスタックと一致するかを判定します。

*※これらのシミュレータクラスを現在の `TowerOfHanoiEnv` と同様のインターフェースで実装し、メイン関数で切り替えられるようにすることで、複雑度 ($N$) に応じたAIの推論崩壊を多角的にモデリングできるようになります。*

## 2. ポテンシャル地形の拡張と力学系の探求
LLMの思考の軌跡をより正確に物理モデルへマッピングするため、数理モデルを拡張します。

* **PotentialPhysicsクラスの拡張**: 現在の1次元の亜臨界ピッチフォーク分岐モデル（$V(x) = -\frac{r}{2}x^2 - \frac{x^4}{4} + \frac{x^6}{6}$）を基礎とし、非線形力学系の知見を用いて数式を高度化します。
* **多次元モデリングと描画**: 論文で指摘された**「Overthinking（簡単な問題で正解付近の無駄な選択肢を探索し続ける現象）」**と、**「Fixation（複雑な問題で初期の誤った推論に固執して抜け出せなくなる推論崩壊）」**を数学的に区別するため、ポテンシャル地形を多次元（例：3次元空間での起伏）に拡張し、`Visualizer` クラスでの描画機能もそれに合わせて改修します。

### ポテンシャルモデルの拡張候補

#### 候補1：カスプ・カタストロフィー（非対称二重井戸ポテンシャル）
* **目的**: 初期の誤りへの「固執（Fixation）」と不可逆な崩壊のモデリング
* **数理モデル**: 
    $$V(x) = \frac{x^4}{4} - \frac{r}{2}x^2 - hx$$
* **パラメータの役割**:
    * $r$: 複雑度パラメータ（谷の数を決める）。
    * $h$: 初期の文脈バイアス（非対称性）。
* **LLM現象とのリンク**: Appleの論文では、推論崩壊の大きな原因として「初期の段階で間違った答えに固執（Fixation）してしまい、軌道修正できなくなる」ことが指摘されています。カスプ・カタストロフィーでは、$h$（初期の間違った推論への傾き）が大きくなると、「正解」側の谷が完全に消滅します。これにより、「いくら思考トークンを消費しても物理的に正解へたどり着けない（完全な推論崩壊）」という相転移を極めて美しく表現できます。

#### 候補2：ラフ・ポテンシャル（多数の局所解を持つ2D地形）
* **目的**: 「過剰思考（Overthinking）」のモデリング
* **数理モデル**: 
    $$V(x,y) = \frac{1}{2}(x^2 + y^2) - A\cos(kx) - B\cos(ky) + C \cdot (\text{Complexity})$$
* **地形のイメージ**: 大きなすり鉢状の地形の中に、無数の小さな「デコボコ（浅い谷）」がある状態。
* **LLM現象とのリンク**: 論文では、簡単な問題において「LLMは早い段階で正解を見つけているのに、無駄な別の解法を探索し続ける（Overthinking）」現象が確認されています。これを2次元空間のラフ・ポテンシャルに拡張すると、粒子が**「無数の浅いデコボコ（冗長な思考経路）に引っかかりながらウロウロするが、巨視的には正解の谷底に向かっている」**状態として表現できます。複雑度が上がるとデコボコの1つが「底なし沼」に変化し、そこに捕まることで相転移（崩壊）が起きます。

#### 候補3：ランジュバン方程式とクラマース遷移（ノイズ誘起脱出）
* **目的**: サンプリング温度（Temperature）と自己修正能力のモデリング
* **数理モデル**: 
    $$\frac{dx}{dt} = -\frac{dV}{dx} + F_{LLM} + \sqrt{2D}\xi(t)$$
* **パラメータの役割**:
    * $D$: サンプリング温度（ノイズの大きさ）。
* **LLM現象とのリンク**: 論文の追加実験において、**「温度を0（Greedy生成）にするよりも、温度を1（サンプリングあり）にした方が、推論崩壊をわずかに遅らせることができる（例：ハノイの塔でN=8がN=9まで延びる）」**という非常に興味深い結果が示されています。これは物理学における「クラマースの脱出問題（Kramers escape）」そのものです。谷（間違った推論）に落ちた粒子が、熱ノイズ（サンプリングによるランダム性）のおかげでポテンシャルの壁を越えて正解側へ脱出できる確率を $\exp(-\frac{\Delta V}{D})$ としてモデル化できます。ノイズ $D$ と複雑度（壁の高さ $\Delta V$）をパラメータにすることで、正確な相図を描くことができます。

#### 候補4：サドルノード分岐（折り目カタストロフィー）
* **目的**: 「思考トークンの減少」を伴う突然の崩壊（Sudden Collapse）
* **数理モデル**: 
    $$V(x) = \frac{1}{3}x^3 - rx$$
* **パラメータの役割**:
    * $r$: 限界値（推論能力と問題の複雑さのバランス）。
* **LLM現象とのリンク**: 論文における最大の謎の一つが、**「ある複雑度を超えると、推論の限界に達するだけでなく、LLMが思考自体を諦めて思考トークンが急減する」**という現象です。サドルノード分岐では、パラメータ $r$ が臨界点を超えた瞬間に、粒子がとどまっていた「安定な谷（思考を続ける状態）」が突然消滅し、無限の奈落（推論放棄）へ転がり落ちます。この「徐々に悪くなる」のではなく「限界を超えた瞬間に突然システムが破綻する」という非連続的な相転移を表現するのに最適です。

## 3. 定量評価・法則の発見（相図の作成）
物理モデルを用いて、LLMの推論崩壊に関する定量的なスケーリング法則を導き出します。

* **バッチシミュレーション基盤の構築**: アニメーション描画（`Visualizer`）をオフにし、パラメータ（複雑度 $N$、試行回数、サンプリング温度などのノイズ）を自動で振って多回数実行・データ収集する測定モードを実装します。
* **測定指標の定義と記録**: 各試行において、推論が崩壊した瞬間（最初の失敗手が発生した位置）、最終的な正答率、消費した思考（Thinking）トークン数を記録します。
* **相図（Phase Diagram）の作成**: 収集したデータに基づき、「低複雑度（非推論モデル優位）」「中複雑度（推論モデル優位）」「高複雑度（双方の完全な推論崩壊）」という3つのレジーム間の相転移現象をマッピングし、複雑度の上昇に対して思考トークンが逆に減少するスケーリングの限界点を特定します。

### 相図（Phase Diagram）の作成に向けて
上記のモデルを組み合わせ、以下のような軸で相図を作成することが目標になります。
* **X軸**: 問題の複雑度（ポテンシャルの壁の高さや非対称性）
* **Y軸**: LLMの推論能力（外力 $F_{LLM}$） または サンプリング温度（ノイズ $D$）

この相図上で、**「非推論モデル優位相（Regime 1）」「推論モデル優位相（Regime 2）」「完全崩壊相（Regime 3）」**の3つの領域が境界線（相境界）で明確に分かれることが証明できれば、ブラックボックスだったLLMの推論スケーリングを、非線形物理学の法則として説明する画期的なアプローチになります。

## 4. ポテンシャルに基づくLLMの推論制御モデル
「推論崩壊の可視化」から一歩踏み込み、シミュレータ側からLLMの推論を自己修復させる制御機構を実装します。

* **LLMReasonerクラスへの介入機能**: 現在のストリーミングによる受動的な思考プロセス取得に加え、API経由で動的にプロンプトを注入（レスキュー）する双方向通信インターフェースを実装します。
* **レスキューフォース（強制的軌道修正）の実装**: `PotentialPhysics` の計算において、粒子が不正解側の深い谷（例：$x \approx -1$）に一定時間留まったことをトリガーとし、モデルが「Fixation（推論崩壊）」に陥ったと判定します。
* **フィードバックループの構築**: 崩壊を検知した瞬間に、LLMに対して「現在の推論軌道は誤っているため、○手前までバックトラックして別の解を探索せよ」といった外力（強制プロンプト）を与え、物理的に粒子を谷底から引き上げて正解軌道へ戻す制御システムを構築します。
