LLM推論崩壊のポテンシャル井戸シミュレータ 開発仕様書
1. プロジェクト概要
Appleの研究論文に着想を得て、大規模推論モデル（LRM）が複雑な問題を解く際の「思考の軌跡（Thinking traces）」を、非線形力学系におけるポテンシャル井戸 V(x) を運動する粒子の軌道としてモデリングし、リアルタイムアニメーションとして可視化するPythonスクリプトを作成する
2. システム要件
言語: Python 3.12
ファイル構成: llm_physics_sim.py の1ファイルで完結させる。
主要ライブラリ: 主要ライブラリ: google-generativeai (Gemini API用), numpy (数値計算), scipy (微分方程式), matplotlib (アニメーション描画)
3. 全体アーキテクチャ（クラス設計）
1ファイル内に以下の4つのクラスを実装し、メイン関数で結合する。
① TowerOfHanoiEnv クラス（パズル環境と状態評価）
役割: Appleの論文でも用いられた「ハノイの塔」の環境シミュレータ
。問題の複雑度（円盤の数 N）を管理する。
機能:
__init__(self, N): 円盤数 N で環境を初期化。
get_prompt(): LLMに与える初期プロンプト（ルールと初期状態・目標状態）を生成する。
evaluate_state(self, current_moves): LLMが生成した手（moves）のリストを受け取り、現在の盤面状態を計算。目標状態への近さを -1.0 (完全に不正解/行き詰まり) から 1.0 (正解) のスコアで返す。
② LLMReasoner クラス（LLM APIハンドラ）
役割: LLM（例: gemini-2.5-flash）と通信し、思考プロセスをステップごとに取得する。。
機能:
LLMにハノイの塔のプロンプトを与え、推論（Chain-of-Thought）を開始させる。
推論プロセスをストリーミングで取得し、一定トークンごと、あるいは新しい「手（move）」が生成されるたびに、その時点での解の候補を抽出する
。
③ PotentialPhysics クラス（力学系とポテンシャル計算）
役割: LLMのスコアを物理現象に変換する。亜臨界ピッチフォーク分岐のモデルを応用する
。
数式モデル:
粒子の状態 x∈[−1,1] は「思考の正しさ」を表す。
ポテンシャル関数V(x) = -(r/2)*x^2 -(x^4)/4 + (x^6) /6 
パラメータ r は「問題の複雑度」に反比例する制御パラメータ。N が小さい（簡単）と x=1 側に深い井戸ができ、N が大きい（複雑）と x=−1（不正解への固執）の井戸が深くなるように設計する
。
状態更新式:  
dx/dt = -dV/dt + F_LLM + noise
​
F_LLM は、TowerOfHanoiEnv から得られたLLMの現在のスコアに基づく外力。LLMが正解に近づけば粒子を x=1 へ引っ張り、間違えれば x=−1 へ引っ張る。

④ Visualizer クラス（リアルタイムアニメーション）
役割: matplotlibの FuncAnimation を用いて、ポテンシャル地形と粒子の運動をアニメーション化する
。
描画要件:
上部パネル: x を横軸、V(x) を縦軸としたポテンシャルの曲線。その曲線上をLLMの現在の状態を表す点（粒子）が動く
。
下部パネル: 時間 t を横軸、状態 x を縦軸とした時系列の軌道（Trajectory）グラフ
。
粒子が x≈1 付近で小さく揺れる様子で「Overthinking」を表現し、x≈−1 の深い谷に落ちて出られなくなる様子で「Fixation（崩壊）」を視覚化する
。

4. メイン処理の流れ (main 関数)
ユーザーに複雑度 N (例: 2〜8) を入力させる。
N に基づき、TowerOfHanoiEnv と PotentialPhysics (パラメータ r を設定) を初期化。
Visualizer の描画ウィンドウを起動。
バックグラウンドの別スレッドで LLMReasoner を走らせ、ストリーミングで思考過程を取得。
思考が更新されるたびに evaluate_state でスコア化し、PotentialPhysics に外力を与えて微分方程式を1ステップ進める。
Visualizer が最新の x の位置を取得し、画面を更新する
